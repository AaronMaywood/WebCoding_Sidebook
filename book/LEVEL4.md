## ボックス

`CSS`で扱うものは全て「ボックス（４角形の箱）」であるとされており、ボックスは`CSS`の重要な基本概念です。

% 例えば、`CSS`でのレイアウトも、これらボックスをどう並べるか？という考えで成り立っています。

ウェブサイトを構成するさまざまなデザイン要素は、例え素材の形が○や☆だったとしても四角のなかに収め、四角としてレイアウトされます。

```{hint}
レイアウト要素の全てが四角だということは、漫画のコマの扱いに似ています。漫画でも様々な形の絵が（基本的には）四角のコマに収められ、コマ割り（レイアウト）されます。
```

`CSS`のボックスの特徴の一つは、ボックスの中に子供のボックスを入れ子にできることです。そして、親のボックスを移動すればその中にある子もついてくるようになっています。この入れ子構造の性質を利用して、親のボックスでページの大規模構造を作りつつ、そのボックスの家族内で子供のボックスを配置して細部を詰めるという分割統治の考え方が可能です。

CSSはボックスの集合体であると説明しましたが、 ボックスが関わる話題を２つに分けることができます。
１つ目がボックスをレイアウトする場合の話題で、２つ目がボックスが共通に持つ内部構造についてです。

順番に説明します。この両方の特徴を押さえましょう。

### ボックスの話題１. レイアウトの方法 ... 通常のフロー＋その他のレイアウト方法

% 参考資料
% - レイアウト一式
% 	https://developer.mozilla.org/ja/docs/Learn/CSS/CSS_layout
% - レイアウトと包含ブロック
% 	https://developer.mozilla.org/ja/docs/Web/CSS/Containing_block
% - 通常のフロー
% 	https://developer.mozilla.org/ja/docs/Learn/CSS/CSS_layout/Normal_Flow

`CSS`でのレイアウトの考え方は、既定の「通常のフロー（flow,流れ）」と、それ以外のレイアウト方法（`float`、`position`、`flex`、`grid`など）の２段構えです。

まず基本の「通常のフロー」を説明します。

「通常のフロー」は既定のレイアウトであり、そのルールは「（インライン要素である）行は横 {bdg-primary-line}`→` に流れ」「（ブロックレベル要素である） 段落は縦 {bdg-primary-line}`↓` に流れる」となっています。

````{hint}
ブロックレベル要素、インライン要素は`CSS`での基本のレイアウトルールである「通常のフロー」を構成します。

```{tip}
なお、ブロックレベル要素やインライン要素は`HTML`寄りの用語です。（「要素」という用語は`HTML`のものでした）
`CSS`で扱うのは（要素ではなく）全て「ボックス」であるため、「ブロックレベルボックス」「インラインボックス」のように用語が変化します。

なお、用語が増えることによる混乱を避けるため本書では「ブロックレベル要素」「インライン要素」の呼び名で統一します。
```
````

````{hint}
通常のフローの正体は、横書きにする**欧文**のスタイルを実装したものです。
日本でも欧文の横書きスタイルを採用しているため、この流れはごく自然なものです。
```{tip}
**世界には様々な書字方向がある**

しかし、世界には様々な書字方向を持つ文化が存在します。彼らにとってみれば、「通常のフロー」の文章は読めたものではありません。
- アラビア語のように横書きで、行の流れが反対（左←右）の文字
    - 例）[WikipediaのCSSのアラビア語ページ](https://ur.wikipedia.org/wiki/%D8%B3%DB%8C_%D8%A7%DB%8C%D8%B3_%D8%A7%DB%8C%D8%B3)
- 中国語や日本語の「縦書き」文字
	- 例）[縦書きWeb普及委員会のページ](https://tategaki.github.io/)

そこで様々な書字方向をサポートする`writing-mode`という`CSS`があります。`writing-mode`を用いれば「通常のフロー」をカスタマイズ（たとえば、行の流れを {bdg-primary-line}`←` にしたり、段落の流れを {bdg-primary-line}`↑` にしたり）できます。
- [writing-mode](https://developer.mozilla.org/ja/docs/Web/CSS/writing-mode)
```
````

「通常のフロー」はボックスが横 {bdg-primary-line}`→` と縦 {bdg-primary-line}`↓` に並びたがる（決してこの逆ではありません）ことを説明する重要なルールで、CSSレイアウトの基本中の基本です。

次に「通常のフロー」以外のレイアウト方法を説明します。

% 素材をどのように配置するかを考えることがレイアウトを考えることに相当し、デザイナーの中で繰り返し検討される自然な考えです。
% デザイナーが考えるように、`CSS`でも豊かなレイアウト表現ができなくてはなりません。

時代とともに豊かなレイアウト表現が可能になりましたが、最初からそうだったわけではありません。
少し`CSS`の歴史を振り返ってみます。

`CSS`が誕生した当時、ウェブは研究者同士が論文のような文章を発表しあう場でした。したがって、文書の表現能力は論文を記述できる程度（ワープロの表現力程度）あれば十分とみなされていました。（「ウェブデザイン」という概念はありませんでした）

この最初期に生まれたのが「通常のフロー」と`float`と`position`です。

`float`は文章の途中に挿し絵を挿入するために生まれました。**挿し絵を挿入し、その周囲にテキストが回り込むというレイアウト方法**です。

`position`は**位置指定**という意味で、ボックスの位置を指定する方法です。ボックスの位置指定には主に２種類あり、**画面上の位置を直接指定する絶対位置指定**`position:aboslute;`と、**一度「通常のフロー」でボックスを配置するが、そこからどのくらいずらすかを指定する相対位置指定**`position:relative;`です。

また位置指定されたボックスは他のボックスに重なる可能性が出てきます。 その際に**どちらを手前に表示するかという制御**(`z-indexプロパティ`)も行えるようになっています。
（`z-index`は`Photoshop`のようなグラフィックツールに代表される「レイヤー」に似た前後方向の重なりを表現します。）

これら`float`、`position`を高度に応用すると、本来の使い方から離れた様々なレイアウトを行うことができました。
このことを発見した当時のデザイナーやコーダー達が段組みなどの高度な表現を試みるようになり、ウェブの表現は当初の論文からかけはなれた、いわゆる「ウェブデザイン」の分野が切り開かれました。

この時代の`CSS`はパズルのように難解なものになり、その灰色の時代は長く続きましたが、ウェブデザインの発展に伴って`float`に代わる**本物の**レイアウトの道具が必要であると広く認識されるに至りました。

そうして登場したのが`flex`や`grid`です。
```{tip}
あまり有名ではなくテキストでも扱いませんが、[`columns`というレイアウト](https://developer.mozilla.org/ja/docs/Learn/CSS/CSS_layout/Multiple-column_Layout)もあります。
% MEMO `columns`は何のため？
% 読みやすさの観点から、行内の文字数は50〜75文字以内という知見があります。
% これを実現するのが`columns`です。
```

`flexbox`は横幅に対して、その中にどのようにボックスを配置するか（中央寄せなのか、余白を均等割にするかなど）を表現するレイアウト方法です。文章の改行と同じように横幅を使い切った時にボックスを改行することもできます。
```{hint}
`flexbox`は行の中にどう文字を配置するのかという問題をボックスに置き換えたものだと言えます。行内の文字のあしらい同様、中央寄せや改行などを扱います。（さらに`flexbox`特有の配置機能もあります）
```

`grid`は`flexbox`と似た面がたくさんありますが、`flexbox`が横方向だけを扱うのに対し、`grid`は「表（`table`）」のように縦横のマス目への配置を扱います。ウェブページの大まかなレイアウトは新聞や雑誌のように縦横のマス目を想定し、そのどの部分にコンテンツを割り付けるかを考えます（=グリッドシステム）。その考えを表現するのが`grid`というレイアウト原則です。

`flexbox`と`grid`の登場によって`CSS`は現代的なウェブデザインのデザインワークに耐えうる表現能力を持ちました。

それまでは`float`を裏技的、パズル的に駆使して同等のことを行っていましたが、もはやその必要はありません。
（ただし、過去に作成されたページを保守する仕事があるため、業務によっては`float`の高度な知識は未だ必要です。）

そして`float`は本来の使い方（文章に挿し絵を配置する）に戻り、シンプルな用法のみで良くなりました。

### ボックスの話題２. ボックスの内部構造である「ボックスモデル」

ボックスのもう一つの側面は、それぞれのボックスには共通する内部構造（=**ボックスモデル**）を持つ、というものです。

全てのボックスには共通する玉ねぎのような階層構造があり、それによってボックス自身の大きさが決定され、また隣り合うボックス同士の余白を決めます。

```{figure} images/gyazo/5dd4d8f5939d9741d0f1cc7b38df2941.png
ボックスには玉ねぎのような階層構造があります。（玉ねぎは<a href="https://unsplash.com/ja/s/%E5%86%99%E7%9C%9F/%E7%8E%89%E3%81%AD%E3%81%8E?utm_source=unsplash&utm_medium=referral&utm_content=creditCopyText" class="reference external">Unsplash</a>の<a class="reference external" href="https://unsplash.com/@k8_iv?utm_source=unsplash&utm_medium=referral&utm_content=creditCopyText">K8</a>が撮影した写真）
```

ボックスの構造は内側から外側に向かって「コンテンツ（内容）」「パディング（境界線とコンテンツの間に敷かれるボックス内部の余白）」「ボーダー（境界線）」「マージン（他のボックスとの余白）」と４階層に渡って広がります。

```{hint}
「マージン」のみがボックスの外側の構造で、それ以外は全てボックスの内部構造です。
```

最も内部に位置する「コンテンツ」の大きさは、親のボックスの大きさを基準に決まりますが（入っている文字などの内容の大きさで決まる場合もあります）、CSSの`widthプロパティ`（幅）や`heightプロパティ`（高さ）で指定することもできます。（→その結果としてコンテンツがはみ出る場合が生じ、はみ出たコンテンツの扱いを決める`overflow`プロパティがあります。）

### 「標準モード」とは
{bdg-dark-line}`テキスト：P.114`

ここに出てくる「標準モード（標準ボックスモデル）」という用語を補足します。
「標準モード（標準ボックスモデル）」に対する言葉は「代替ボックスモデル」です。

この両者は、次の節に登場する`box-sizing`プロパティで切り替えることができます。
`box-sizing: border-box;`を設定したときが「代替ボックスモデル」で、設定しない時（初期値の`box-sizing: content-box;`の時）が「標準ボックスモデル」です。

「標準ボックスモデル」と「代替ボックスモデル」の違いは、幅`width`（及び高さ`height`）の解釈の違いです。

```{figure} images/gyazo/331926bbb851e57ea80e704659a3d8d2.png
:class: full-width
代替ボックスモデルは、標準ボックスモデルの欠点を取り除いてわかりやすくしたものです。
```

幅`width:100px;`を指定した場合で説明します。
標準ボックスモデルでは、幅`width:100px;`は、「内容（コンテンツ）」の幅のみを指します。
このボックスに`padding: 50px;`、`border: 5px solid black;`（太さ5pxの境界線）が指定されている場合は、ボックス全体の幅は`100+50+5=155px`となります。

それに対し、代替ボックスモデルでは幅`width:100px;`が「内容（コンテンツ）」と「パディング」「ボーダー」の３つを併せたものを指します。
したがってボックス全体の幅は`100px;`のままです。

さて、この両方の幅の解釈ですが、レイアウトする時に直感的に分かりやすいのは「代替ボックスモデル」の方です。
「代替ボックスモデル」はボックスの大きさを把握するのが簡単です。
このため、`box-sizing: border-box;`をリセットCSSに含めることも行われます。

% ```{code} css
% * {
%   margin: 0;
%   padding: 0;
%   box-sizing: border-box; /* 追加 */
% }
% ```

## margin や padding プロパティの一括設定（ショートハンド設定）
 {bdg-dark-line}`テキスト：P.115`

ショートハンドは複数のプロパティを一括で指定できる簡略表現です。
```{code} css
例)
margin-top: 10px;
margin-right: 10px;
margin-bottom: 10px;
margin-left: 10px;
↓（上記４つのプロパティを一行で表したショートハンド） 
margin: 10px;
```

`margin`と`padding`のショートハンドは同じ書式なので、`margin`を覚えれば`padding`も覚えたことになります。

## ボーダーの作例
{bdg-primary-line}`CSS教材：12_ボーダー` フォルダを`VS Code`で開き、`lesson1.html`のページに対して{bdg-dark-line}`テキスト：P.116`のようなボーダーを設定してみて下さい。

````{tip}
`margin`や`padding`はショートハンドだと紹介しましたが、ここで説明する`border`もまたショートハンドです。
（`border`は`border-width`、`border-style`、`border-color`の３つのプロパティのショートハンドです。）
```{code}
border-width: 2px;
border-style: solid;
border-color: green;
↓（ショートハンド）
border: 2px solid green;
```
````

````{tip}
`CSS`ではショートハンドはありふれたもので、`background`、`font`、`box-shadow`など至るところにあります。

**ショートハンドを利用する際の注意点**を説明します。

マージンを例に説明します。マージンを上部のみ`50px`とし、その他は`0px`にしたいとします。
このように書いてはなりません。
``` css
body {
  margin-top: 50px;
  margin: 0;
}
```
これは、`margin: 0;`が後に書いてあり、かつ`margin: 0;`は`margin-top: 0;`の意味も含むため、上書きされてしまうからです。

したがって、このようにして下さい。
``` css
body {
  margin: 0;        /* ショートハンドを先に書き、 */
  margin-top: 50px; /* 個別のプロパティを後に書く */
}
```
こうすれば`margin: 0;`の指定を`margin-top: 50px`が上書きすることになります。

ここで挙げたマージン以外のショートハンド全てに共通する性質です。
````

## ボックスの外部余白（margin）の相殺（そうさい）について

テキストを少々飛ばして、{bdg-dark-line}`テキスト：P.137〜138` {bdg-primary-line}`CSS教材：23_相殺について` の話題を取り上げます。

これは隣り合ったボックス同士にマージンが設定されている場合の扱いに関する話題です。

**隣り合ったボックスのマージンの量は、両方のマージンの足し算ではなくどちらか長い方が採用されるというルールがあり、これをマージンの相殺と言います**。

**相殺**があるのは`margin`だけです。`border`、`padding`にはありません。

相殺という言葉が難しいのでたとえ話で説明します。

一軒家があったとします。その一軒家の持ち主Aは、隣の家との間隔（マージン）を`5m`は欲しいと思っていました。
そこへ隣の家を建てることになり、その隣家の持ち主Bも隣の家との間隔を最低`5m`にしたいと思っていました。
この時、二軒の家の間を**5m**にすれば両者の希望を満たせます。

数値だけをおいかければ`5m+5m=10m`とも思ってしまえますが、「余白」の性質から`5m`で十分だとなります。
この性質を（数値上は相殺されているように見えるため）「マージンの相殺」と呼んでいます。

**相殺は両者のマージンのうち長い方を採用するという意味です。それだけの話です。**
もし隣家の持ち主Bが`7m`の間隔が欲しいと思った場合には、両者の要求する間隔(Aは`5m`を希望, Bは`7m`を希望)のうち、長い方(`7m`)を選択します。

ボックスモデルを構成する「マージン」「ボーダー」「パディング」「内容（コンテンツ）」のうち、相殺があるのは隣家との間隔を決めるマージンだけです。
その他の「ボーダー」「パディング」「内容（コンテンツ）」は家の内部の話であり、隣接するボックス同士の関係である相殺は関与しません。
