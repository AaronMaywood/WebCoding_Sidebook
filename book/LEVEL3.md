## セレクタ―

単純な`CSS`を書く体験を済ませたところで、ここからは`CSS`を構成する細かな部分の学習を進めていきます。
最初に学ぶのが「セレクター」です。

````{glossary}
セレクター
  セレクターは「セレクト（選択）する道具」と言う意味で、`h1 { color: green; }`（「見出し１の文字色は、緑です。」）と書いたときの`h1`（「見出し１の」）の部分、つまり「**主語**」**の役割を果たすもの**です。

プロパティ
  セレクターに示された部分に対し、具体的なスタイル指示がプロパティです。
  ```{tip}
  「`color: green;`」が一つのプロパティで、宣言とも言います。また、`color`の部分をプロパティ名、`green`の部分をプロパティ値と言います。
  % 宣言(Declaration)はここに説明がある https://developer.mozilla.org/ja/docs/Learn/Getting_started_with_the_web/CSS_basics
  ```
  ```{hint}
  英単語のプロパティ（`property`）には「（ものの）特性」という意味があります。
  % https://ejje.weblio.jp/content/property
  ```
````

（`CSS`は「主語」に相当する「セレクター」と、具体的なスタイル指示である「宣言（プロパティとその値）」からなります。）

##### 名前を付ける機能について

`HTML`の要素には製作者が名前を付けることのできる機能が２種類あります。

1. まず、{bdg-dark-line}`テキスト：P.82` で説明する「クラス名」です。（「[](div-span)」でも紹介しました）

	クラスは「学級（生徒をグループ分けしたもの）」と同じ意味で、要素をグループ分けするのに用います。
	次のように書いたら、

	```html
	<h1 class="a">見出し１</h1>
	<p class="a">段落１</p>
	<p class="b">段落２</p>
	<p>段落３</p>
	<h2 class="a">見出し２</h2>
	<p class="b">段落４</p>
	```

	「aクラス」に所属するのは「見出し１」「段落１」「見出し２」で、「bクラス」に所属するのは「段落２」「段落４」です。（「段落３」はどのクラスにも所属していません。）

2. 次に、{bdg-dark-line}`テキスト：P.83` にある`ID`名です。

	% https://www.weblio.jp/content/%E3%82%A2%E3%82%A4%E3%83%87%E3%82%A3%E3%83%BC
	`ID`は`identification`の略で、俗に「IDカード」などという個人を特定する身分証明書を意味する単語です。
	名前を付けるという意味ではクラスと同じですが、クラスが複数の要素に同じ名を付けてグループ分けすることができたのとは対象的に、個の識別名である`ID`は同じ名前を同じページ内で２度以上使うことができません。

	```html
	<p id="a">段落１</p>
	<p id="a">段落２</p> <!-- 同じ名前は２度使えません！ -->
	<p id="b">段落４</p>
	```

	`ID`名は常に特定の一つの要素を指す名前として使用します。

##### スタイルの優先順位について

{bdg-dark-line}`テキスト：P.82` には、「HTML内で同じクラス名を複数のあちこちで使用可能で、通常要素（タイプセレクタ）の**約１０倍強く作用します**。」という説明があります。これは「スタイルには優先順位がある」ということを説明しています。

次の`HTML`をもとに説明します。
```HTML
<p>段落１</p>
<p class="test">段落２</p>
```

次の`CSS`では、`p`要素の色は`green`となります。（最初の`red`の指定は次の`green`で上書きされます。つまり、`green`が優先します。）
```CSS
p {
  color: red;
}

p {
  color: green;
}
```

`CSS`では基本的に**同じセレクター`p`に対する同じスタイル`color`があったとき、後の方に書いたスタイル**が優先されます。

しかし、クラスセレクタを付与すると「約10倍強い」優先度となり、`class="test"`が指定された「段落２」の文字色は`red`となります。（「段落１」にはクラス名がないので`green`のままです。）
```CSS
/* "p.test" とは、p要素かつクラス名が"test"であるものを指します。なお、".test" というようにクラス名だけを指定することも可能です。 */
p.test {
  color: red;
}

p {
  color: green;
}
```

これが`CSS`の優先順位です。

優先順位のおおざっぱな理解は次のとおりです。
- 通常の要素（例えば`p`）よりも、わざわざ製作者がクラス名を付けた要素の方が優先度が高くなるようになっています。（ありふれた`p`は`green`だが、`testクラス`に所属している`p`は特別に`red`というように）
  - 更に、`ID`名は固有名称であるため、最も優先度が高くなります。（ありふれた`p`は`green`、`testクラス`に所属している`p`だけは`red`、しかし`taro-yamada`（固有名称だと思って下さい）だけは`blue`のように）

#### 子孫セレクタ
 
テキストの後の方（{bdg-dark-line}`テキスト：P.134`）に出てくる「子孫セレクター」は良く使用するので合わせて説明します。

子孫セレクターは、親の特徴を手がかりにし、その子を対象にするものです。

具体例で紹介します。

次の`HTML`のうち、「段落１」だけ背景色を変えたいとします。
```html
<div class="a">
  <p>段落１</p>
</div>
<div class="b">
  <p>段落２</p>
</div>
```

次の`CSS`（`p`セレクター）では、「段落１」「段落２」の両方を指定してしまっています。
```css
p {
  background: red;
}
```
 
「段落１」の`HTML`上の特徴は、親が`div class="a"`であることです。（それに対し、「段落２」の親は別のクラス`b`に所属しています。）
そこで、「段落１」を指定するために、「クラス`a`」を手がかりにし、「クラス`a`の子`p`」と書きます。

```css
/* クラス`a`の子`p` */
.a p {
  background: red;
}
```

この時、親`.a`と子`p` の間に、**空白文字が必要です**。この空白文字こそ、**子孫セレクター**を表しています。

```{hint}
この空白文字を「〜の中の子供〜」と翻訳すると分かりやすいです。
`.a p`は、「クラス`a`の中の子供`p`」とよむことができます。
```

子孫セレクターに似て大変紛らわしいのが、次の表現です。
```css
div.a {
  background: red;
}
```

ここでのセレクターには、`div`要素とクラス`a`の間に**空白文字がありません。この場合には、「〜かつ〜」と読みます。つまり、この例では「`div`要素かつクラス`a`」となります**。

セレクターの間に空白文字が一つでもあれば子孫セレクター「〜の中の子供〜」となり、空白文字がなければ「〜かつ〜」と読み分けるようにして下さい。

````{tip}
なお、「子孫セレクター」は「子」だけでなく「孫」（及びそのまた孫...）も含みます。次の`HTML`に対し、セレクター`.a a`を使用ると、`<a href="">リンク</a>`を指します。

```html
<div class="a">
  <p>段落１です。ここに<a href="">リンク</a>があります。</p>
</div> 
```
````

````{admonition} 応用
{bdg-dark-line}`テキスト：P.134`に出てくる「子孫セレクター」の表現は`nav.Atype ul li a`です。これは次の`HTML`のどの部分を指しているでしょうか？
```html

  <ul>
    <li><a href="#">項目１</a></li>
    <li><a href="#">項目２</a></li>
    ...
  </ul>
</nav>
```
```{dropdown} 答え合わせ
`nav.Atype ul li a`は「`nav`かつ`.Atype`（つまり`<nav class="Atype">`）の中の子`ul`の中の子`li`の中の子`a`」と読みます。
これは上の`HTML`中の全ての`a`要素を指しています。
```
````

{bdg-dark-line}`テキスト：P.85〜87` は、どこに`CSS`を書くか？という話題です。
ウェブサイトでは、ウェブサイトを構成する複数のページ（トップページや下層ページ）の統一性を持たせるために、トップページに使用したデザインを踏襲して下層ページを作成します。これはトップページ用に書いた`CSS`を**使いまわしできる**ということです。したがって{bdg-dark-line}`テキスト：P.87`の「(C)外部ファイル記述法」が主流であるという結論です。

（内部記述（(A)インラインCSS、(B)`head`内記述）の２つを使用するシーンは学習や実験の用途です。さっと書いて試せるため、試しに入れてみるといった用途に便利です。）

```{hint}
なお、{bdg-dark-line}`テキスト：P.95` に言及がありますが、外部`CSS`の場合には冒頭に`@charset "utf-8";`の一文を書きます。
この一文を書く事によって`CSS`に日本語のコメントを書くことができるようになります。
（逆に言えば、日本語のコメントを書かない場合には`@charset "utf-8";`は不要です。）
```

(why-62.5)=
````{hint}
**高度な話題**

`html { font-size: 62.5%; }`とは何か、なぜ今は推奨とされないのかについての説明をします。

まず、`html {font-size:62.5%;}`を使用する方法がなぜ生まれたかを説明します。

1. 単位`px`のかわりに単位`rem`を使用すると、ブラウザの「規定のフォントサイズの設定変更」に対応することができます。

	（ {bdg-dark-line}`テキスト：P.100〜103` の「■実習：絶対値「px」と、相対値「rem」「em」の違いとは？」を参照のこと）

	ブラウザの規定のフォントサイズの変更は、ウェブサイトの文字が小さくて読みにくいと感じている人のためのもので、ブラウザーの設定画面から行うことができます。
	
	```{tip}
	ブラウザーの規定のフォントサイズは以下の手順で変更可能です。

	> Google Chromeのフォントをカスタマイズする方法
	> https://pc-karuma.net/how-to-customize-google-chrome-fonts/
	```

	単位`rem`はブラウザの規定のフォントサイズからの相対値を指定しているため、ブラウザーの規定のフォントサイズの変更に連動できます。それに対し、絶対的な単位である`px`は、ブラウザーのフォントサイズ設定とは独立しているため、規定のフォントサイズ設定は無視されます。

	従って、`px`ではなく`rem`を使用しましょう。

2. デザイナーからの指示はほとんどの場合`px`です。`rem`を使用するにあたり、`px`から`rem`への変換が必要です。

	規定では`1rem=16px`という関係であるため、たとえば`18px`にしたい場所に「何`rem`」になるかを計算しなくてはいけません。（具体的には「18/16=1.125rem」という計算が必要です。）

3. この計算の手間を楽にするために `html { font-size: 62.5%; }` があります。

	これにより`1rem=10px`という関係になるため、計算が簡単になります。`18px`にしたいときには`1.8rem`と書くだけです。

単位`rem`を使用すること、`rem`の使用を楽にするために`html { font-size: 62.5%; }`を使用することの理由は、全て「`1.`」のため（ブラウザの「規定のフォントサイズの設定変更」に対応するため）にあります。

**時代の変化**

しかし、現代のブラウザーはわざわざ「規定のフォントサイズ」の設定を変更しなくとも、手軽にズームできる機能が付くようになりました。

Google Chrome ではショートカットキー {bdg-dark}`Ctrl + マウスホイール` の操作で拡大縮小できます。

この「ズーム」の機能は画面全体を拡大するためフォントサイズに`px`が使用されていても問題なく拡大縮小できます。

このため「`1.`」へ対応するメリットが小さくなりました。

```{tip}
この手軽な「ズーム」機能があればブラウザーの規定のフォントサイズの設定はいらないのではないか？

実際にブラウザーの一つ、`Safari`では「規定のフォントサイズ」の設定項目自体が無くなっています。

これは「フォントサイズ」の設定変更より「ズーム」の使い勝手がよいこと、また、`macOS`側のアクセシビリティ機能で`OS`全体のフォントサイズを大きくできることから、ブラウザーにフォントサイズの調整機能が不要と判断されたためと考えられます。
```

したがって、単位`rem`を使用すべき技術的な理由はなくなっています。

そこから単位`px`や`rem`の使用は自由となりました。どちらの単位を使用しても問題ありません。
一方で、`px`や`rem`の使用は**適材適所が求められる**ということでもあります。
どちらが良いかを判断するには経験が必要であるため、これは中級者の課題です。
````

## 色

`CSS`で色を指定するにはいくつかの方法があります。

1. `red`, `green`等の色名を使う
2. 光の３原色(RGB=赤、緑、青)で指定する
  a. 16進法を使って指定する
  b. 10進法を使って指定する→`rgb()`を使います→併せて透明度も指定できる`rgba()`もあります
3. （テキストの範囲外ですが、`rgb()`の他にグラフィックツールでお馴染みの色相・彩度・明度で色を作ることのできる`hsl()`を使用できます。`hsl(0, 100%, 50%)`と書けば色相0°・彩度100%・明度50%で赤となります。[`hsl()`についての記事](https://developer.mozilla.org/ja/docs/Web/CSS/color_value/hsl)）
4. （後のテキストで出てくるグラデーションも色の一種です）

ただし、これらの色を作るのは我々コーダーではなく、その前の工程でデザイナーが決定します。
**したがって`CSS`で色を作る方法まで学ぶ必要はなく、デザイナーが指定した色（１６進表現が基本で、「ここの色は`#ff0000`で」などと指示されます）を`CSS`で表現できればよいということになります**。（デザイナーからもらったデザイン設計の画像を画面に表示し、カラーピッカーで１６進数の色番号を拾う方法もあります。いずれにせよ、元になる色の指定はデザイナーの仕事です。）
