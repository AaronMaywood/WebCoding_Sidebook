# LEVEL3(LEVEL3.md) CSS - ボックスモデル

## STAGE11 ボックスモデル

CSSの重要な概念に「ボックス」があります。CSSで扱うものは全て「ボックス（４角形の箱）」であると説明します。

ウェブサイトを構成するさまざまなデザイン要素は、例え素材の形が○や☆だったとしても四角のなかに収め、四角としてレイアウトされます。
レイアウト要素の全てが四角だということは、漫画のコマの扱いに似ています。漫画でも様々な形の絵が（基本的には）四角のコマに収められ、コマ割り（レイアウト）されます。

CSSでのボックスの特徴はボックスの中に子供のボックスを入れ子にできることです。そして、親のボックスを移動すればその中にある子もついてきます。その性質を利用し、親のボックスを用いてページの大規模構造を作りつつ、そのボックスの家族内で子供のボックスを配置して細部を詰めるという分割統治の考え方が可能です。

CSSはボックスの集合体であると説明しましたが、 ボックスが関わる話題は２つに分けることができます。
１つ目がボックスをレイアウトする場合の話題で、２つ目がボックスが共通に持つ内部構造についてです。

順番に説明します。この両方の特徴を押さえましょう。

１つ目の話題がレイアウト（配置）の方法です。
レイアウトには「通常のフロー（flow,流れ）」と呼ばれる既定のルールがあり、その上に`float`、`position`、`flex`、`grid`などを指定してカスタマイズしていきます。

「通常のフロー」はCSSに何も書かない時のデフォルトのレイアウトルールであり、欧文の横書きの書字方向を実装したものです。
具体的には「行は横{bdg-primary}`→`に流れ」「段落は縦{bdg-primary}`↓`に流れる」というものです。

この「通常のフロー」は既定であり日本語圏でも欧文の横書きは自然であるため、通常意識することはありません。（漫画のコマを読む順番を通常意識しないのと同様です。）
しかし「通常のフロー」はボックスが横{bdg-primary}`→`と縦{bdg-primary}`↓`に並びたがる（けっしてこれらの逆ではありません）ことを説明する重要なルールで、CSSレイアウトの基本中の基本です。

**世界には様々な書字方向がある**

- 中国語や日本語の「縦書き」文字

- アラビア語のように横書きで、行の流れが反対（左←右）の文字

`writing-mode`を用いれば「通常のフロー」をカスタマイズすることができます。

さて、ウェブデザインにおいて、デザイナーが扱う重要な概念の一つがレイアウト（配置）です。
素材をどのように配置するかを考えることがレイアウトを考えることに相当し、デザイナーの中で繰り返し検討される自然な考えです。

デザイナーが考えるように、CSSでも豊かなレイアウト表現ができなくてはなりません。

時代とともに、現在では豊かなレイアウト表現が可能になりましたが、最初からそうだったわけではありません。
少しCSSの歴史を振り返ってみます。

CSSが誕生した当時、ウェブは研究者同士が論文のような文章を発表しあう場でした。したがって、文書の表現能力は論文程度あれば十分とみなされていました。
この最初期に生まれたのが「通常のフロー」と`float`と`position`です。

`float`は文章の途中に挿し絵を挿入するために生まれました。**挿し絵を挿入し、その周囲にテキストが回り込むというレイアウト原則**です。

`position`は位置指定という意味で、ボックスの位置を指定する方法です。ボックスの位置指定には主に２種類あり、**画面上の位置を直接指定する絶対位置指定**`position:aboslute;`と、**一度「通常のフロー」でボックスを配置するが、そこからどのくらいずらすかを指定する相対位置指定**`position:relative`です。また位置指定されたボックスは他のボックスに重なる可能性が出てきます。 その際に**どちらを手前に表示するかという制御**(`z-indexプロパティ`)も行えます。 これは`Photoshop`のようなグラフィックツールに代表される「レイヤー」に似た画面の前後方向の多層構造を表現します。

これら`float`、`position`を裏技的に応用すると、本来の使い方から離れた様々なレイアウトを行うことができました。
このことを発見した当時のデザイナーやコーダー達が段組みなどの高度な表現を試みるようになり、ウェブの表現は当初の論文からかけはなれた、いわゆる「ウェブデザイン」の分野が切り開かれました。

この裏技の力を頼りにした灰色の時代は長く続きましたが、ウェブデザインの発展に伴って`float`に変わる本物のレイアウトの道具が必要であると広く認識されるに至りました。
そうして登場したのが`flex`や`grid`です。（あまり有名ではありませんが、`columns`もあります。）

`flexbox`は横幅いっぱいに対して、その中にどのようにボックスを配置するか（中央寄せなのか、余白を均等割にするかなど）を表現するレイアウト原則です。文章の改行と同じ用に横幅を使い切った時にボックスを改行することもできます。

`grid`は`flexbox`と似た面がたくさんありますが、`flexbox`が横方向だけを扱うのに対し、`grid`は「表（`table`）」のように縦横のマス目への配置を扱います。ウェブページの大まかなレイアウトは新聞や雑誌のように縦横のマス目を想定し、そのどの部分にコンテンツを割り付けるかを考えます（=グリッドシステム）。その考えを表現するのが`grid`というレイアウト原則です。

### ボックスの話題２. ボックスの内部構造である「ボックスモデル」

ボックスのもう一つの側面は、それぞれのボックスには共通する内部構造（=**ボックスモデル**）があるというものです。

全てのボックスには玉ねぎのような階層構造があり、それによってボックス自身の大きさが決定し、また隣り合うボックス同士の余白を決めます。

ボックスの構造は内側から外側に向かって「コンテンツ（内容）」「パディング（境界線とコンテンツの間に敷かれるボックス内部の余白）」「ボーダー（境界線）」「マージン（他のボックスとの余白）」と４階層に渡って広がります。
「マージン」のみがボックスの外側の構造で、それ以外は全てボックスの内部構造です。

最も内部に位置する「コンテンツ」の大きさは、そこに入っている文字などの内容の大きさで決まりますが、CSSの`widthプロパティ`（幅）や`heightプロパティ`（高さ）で指定することもできます。（→その場合、コンテンツがはみ出ます。はみ出たコンテンツの扱いを決める`overflow`プロパティがあります）

ボックスの残りの構造「パディング」「ボーダー」「マージン」には、それぞれ`paddingプロパティ`、`borderプロパティ`、`marginプロパティ`が対応します。

これら「ボックスモデル」から、テキストの説明に移ります。次に進んで下さい。

### 「標準モード」とは
{bdg-dark-line}`テキスト：P.114`

ここに出てくる「標準モード」という用語を補足します。

次の節に登場する`box-sizing: border-box`を設定したときを「代替ボックスモデル」と言い、それに対して`box-sizing`を設定しない時（初期値の`box-sizing: content-box`の時）を「標準ボックスモデル」と言います。
標準ボックスモデルのことをここでは「標準モード」と説明しています。

「標準ボックスモデル」と「代替ボックスモデル」の違いは、幅`width`（及び高さ`height`）の解釈の違いです。

幅`width:100px`を指定した場合で説明します。
標準ボックスモデルでは、幅`width:100px`は、「内容（コンテンツ）」の幅のみを指します。
このボックスに`padding: 50px`、`border: 5px solid black`（太さ5pxの境界線）が指定されている場合は、ボックス全体の幅は`100+50+5=155px`となります。

それに対し、代替ボックスモデルでは幅`width:100px`が「内容（コンテンツ）」と「パディング」「ボーダー」の３つを併せたものを指します。
したがってボックス全体の幅は`100px`のままです。

ボックスの大きさを把握するには「代替ボックスモデル」が直感的であるため、リセットCSSに`box-sizing: border-box`を含めることも多いです。

## margin や padding プロパティの一括設定（ショートハンド設定）
 {bdg-dark-line}`テキスト：P.115`

ショートハンドとは短略形という意味で、複数のプロパティを一括で指定できる短縮表現です。
```{code} css
例)
margin-top: 10px;
margin-right: 10px;
margin-bottom: 10px;
margin-left: 10px;
↓（上記４つのプロパティを一行で表した短略形） 
margin: 10px;
```

`margin`の４種類の短略形を覚えることを推奨します。
（`margin`と`padding`の短略形は同じ書き方なので、`margin`を覚えれば`padding`も覚えたことになります。）

## ボックスの外部余白（margin）の相殺（そうさい）について
 {bdg-dark-line}`テキスト：P.137〜138` {bdg-primary-line}`教材：23_相殺について`

隣り合ったボックスに設定したマージン同士の扱いに関する話題です。
余白の性質を考慮し、隣り合ったボックスのマージンの量は足し算ではなく長い方が採用されるというルールがあり、これをマージンの相殺と言います。

相殺という言葉が難しいのでたとえ話で説明します。

% {{TODO}}ソーシャルディスタンスは5mで十分
% 一軒家があったとします。その一軒家の持ち主Aは、隣の家との間隔（マージン）を5mは欲しいと思っていました。
% そこへ隣の家を立てることになり、その隣家の持ち主Bも隣の家との間隔を最低5mにしたいと思っていました。
% この時、二軒の家の間を**5m**にすれば両者の希望を満たせます。

これが「マージンの相殺」と呼ばれるルールです。
両者の希望を足して、5m+5m=10mもの長さを確保する必要はありません。
Aが希望するボックス（家）のマージン5mは、隣人Bのボックスのマージン5mと重なり合わせて（これが"相殺"です）長さを決定します。

別の角度から相殺を説明すると、両者のマージンのうち長い方を採用するという意味になります。
もし隣家の持ち主Bが7mの間隔が欲しいと思った場合には、両者の要求する間隔(Aは5mを希望, Bは7mを希望)のうち、長い方(7m)を選択します。

ボックスモデルを構成する「マージン」「ボーダー」「パディング」「内容（コンテンツ）」のうち、相殺があるのは隣家との間隔を決めるマージンだけです。
その他の「ボーダー」「パディング」「内容（コンテンツ）」は家の内部の話であり、隣接するボックス同士の関係である相殺は関与しません。
