## {octicon}`book` **要素**と**タグ**

%ここまでのことを説明します。

HTMLには文章を構成するさまざまな部品が用意されており、それら部品のことを「**要素**」と言います。

```{math}
:label: html
HTMLにはさまざまな要素がある
```

**要素**の例として、すでに**h1要素**（見出し要素）を見ました。
```{math}
:label: element
h1は見出しを表す要素
```

**h1要素**を作るには、`<h1>〜</h1>`という書き方をします。`<h1>`や`</h1>`の記号を「**タグ**」と言います。

```{math}
:label: tag
要素を表すにはタグを使う
```

````{tip}
タグ([tag](https://ejje.weblio.jp/content/tag))とは洋服に付いているタグと同様にその洋服がどんなものか（その文章が何か）を示すラベルのことです。

```{figure} https://i.gyazo.com/7cc58c91192bcfee3e3893b6ea9f1fe9.jpg
洋服のタグ
```
````

% コンピューターは人間とは違って「見出し１」などの文章を見てもそれが本当に見出しなのかを知ることができませんので、人間が「`<h1>見出し１</h1>`」とタグで括って教えてやる必要があります。
「タグで括って要素を指定する」行為を「タグ付け」や「マークアップ(Markup)」と言います。

% ```{math}
% :label: markup
% マークアップとは、タグを付けること
% ```

```{tip}
> 「マークアップ」という用語は、英語圏で伝統的な出版の作業過程である原稿の「マーキング・アップ」という作業から派生した。「marking up」とは、原稿用紙の余白に印刷に関する指示の記号を書き加えることである。
> ―――https://ja.wikipedia.org/wiki/%E3%83%9E%E3%83%BC%E3%82%AF%E3%82%A2%E3%83%83%E3%83%97%E8%A8%80%E8%AA%9E#%E8%AA%9E%E6%BA%90
```

HTMLは要素の集合体です。
```{hint}
HTMLには要素以外のものはありません。要素が全てです。
```
そして要素はタグによってマークアップされます。

したがって、HTMLで文書を作成するとは、テキストに様々な要素をタグ付けする（マークアップする）ことが全てです。

## {octicon}`book` HTMLの文法 Lv1

HTMLは要素の集合（=タグの集合）だと説明しました。

そのルールはきっちりと決まっています。

最初のルールは{numref}`html-syntax1`に示すように、すでに学んだ「要素の書き方（=マークアップの方法）」です。
```{figure} https://i.gyazo.com/7f2f98f5c5a3bbf84acccf8b81742a52.png
---
name: html-syntax1
---
文法１："内容をタグで囲う"ことで要素を表現します。
```

```{figure} https://i.gyazo.com/e908cd99d28b009a3280498a7e064bf0.png
---
name: html-syntax2
---
文法２：一部の要素は**空要素**と言い内容を持つことがありませんので"開きタグのみで表現"します。
```

本書で扱う空要素には画像を表現する`img`要素と、改行を表す`br`要素の２つです。
空要素に対し閉じタグを書くことは誤りです。
```{tip} HTMLにはエラーがない
HTMLは誤りに対して寛容です。
誤りが合った場合は適宜に解釈してそれなりに表示してくれます。
それなりというのは、大前提としてHTMLは壊れていますので、それをどう解釈し表示されるかはHTMLを処理するブラウザー任せになってしまうということです。

```{math}
文法を間違えるとブラウザ上の処理で誤解を招きます
```

結局のところHTMLは製作者の責任において**正しく***書く必要があります。
```{tip}
どうせ正しく書く必要があるなら、最初から誤りを指摘してくれたほうが却って親切です。
しかし、HTMLはHTMLの専門家でなくとも書くことができるように設計されました。
HTMLの書き手が間違ってもそれなりに表示されたほうが良いだろうということでこのような仕様になっています。
```

```{figure} https://i.gyazo.com/b8c02083386c1c512ae3e84ed6ff182d.png 
---
name: html-syntax3
---
文法３：開きタグには**属性**と呼ばれる設定を埋め込むことができます。要素によって持てる属性が決められています。
```

```{figure} https://i.gyazo.com/38e498492e580156f11b3b3a04069f55.png
---
name: html-syntax4
---
文法４：要素は入れ子にすることができます。囲ったほうを親（親要素）、囲われた方を子（子要素）といいます。入れ子は何重にもできます。要素によっては入れ子ができないものもあります。
```

### 結論
1. ウェブサイト制作では、どのウェブブラウザでも確実に動作するように制作する必要があります。
2. しかし、ウェブブラウザの種類は数え切れないほどあります。現実にはシェア率の高い主要なブラウザーに絞る必要が有ります。

### ウェブブラウザには種類がある

ウェブブラウザには種類があります。
主要なブラウザーだけを取り上げると、Google社の"Chrome(クローム)"、Microsoft社の"Edge(エッジ)"、Apple社の"Safari(サファリ)"、非営利法人 Mozilla の"Firefox(ファイヤーフォックス)"の４つです。


https://ja.wikipedia.org/wiki/Safari
https://ja.wikipedia.org/wiki/Google_Chrome
https://ja.wikipedia.org/wiki/Microsoft_Edge
https://ja.wikipedia.org/wiki/Mozilla_Firefox


ウェブの利用者からみたとき、これらの違いはあまりありません。
特に、肝心のウェブページの表示品質はほとんど同じです。
最初に使い始めて慣れたブラウザーを長年使用しているのが大半のケースだと思います。
明確にブラウザーを選定して使い分けている人は、操作の方法やカスタマイズのしやすさなどのページの表示品質とは別のところで選んでいるでしょう。

なぜ似たようなブラウザーがあるかと言えば、
そもそもは各社シェアの奪い合いがあった
有料版ブラウザーとOS囲い込み
ウェブの健全性に役立っています。
WHATWG
https://ja.wikipedia.org/wiki/Web_Hypertext_Application_Technology_Working_Group

ウェブ制作者にはこれら４つのブラウザーはHTML/CSSの細かな動作に違いがあることがあり、ウェブサイトを出荷する際には必ず４つ全てのブラウザーで動作確認した上で行う必要があります。
また、有るブラウザーで発生した表示崩れを修正するための技術が求められます。
一般の利用者がどのブラウザーでも同様の表示品質に見えるのは、実はウェブサイトの製作者の苦心の結果です。

ウェブサイト製作者は、必要となる検証作業を最小限にするため、どのウェブブラウザーが多く使われているかのシェア（通常は国内シェア）を把握しておく必要があります。

- スマートフォン版のブラウザー Chrome, Safari

主要でないブラウザーを含めるともっとたくさんのブラウザーが存在します。これらは一般的なブラウザーでないため、ウェブ制作社は表示品質を保証する必要がありません。利用者の責任において利用してもらうというスタンスです。

## {{TODO}} フォルダ・ファイル名の名前

https://developer.mozilla.org/ja/docs/Learn/Getting_started_with_the_web/Dealing_with_files
> フォルダ名やファイル名を小文字で書き、スペースを入れず、単語をハイフンで区切る習慣をつけるとよいでしょう。そうすれば、後で問題にぶつかることも少なくなるはずです。
> 大文字や空白を使わない
> 
> 簡単に言うと、ファイル名にはハイフンを使用した方が良いということです。Googleの検索エンジンはハイフンを単語の区切りとして扱いますが、アンダースコアはそのように見なされません。

Windows/Macではフォルダ名・ファイル名には自由な文字が使用できますが、Webサーバーにアップロードする際にファイル名が化けて取扱いできなくなることがあります。
したがってウェブ開発では、フォルダ名・ファイル名に使用できる文字を限定して使用します。

以下にそのルールを示します。

- 半角英数文字とハイフンとアンダースコアのみを使用する
	- アルファベット大文字 AからZ
	- アルファベット小文字 aからz
	- 数字 0から9
	- 記号 `-`、`_`、`.`
- ファイルには拡張子が必須
	- 拡張子はピリオド+ファイルの種類(ファイルの種類毎に定められた3〜4文字)
		- `.html`や`.css`や`.jpg`
- NG! 日本語、/や!などのその他の記号
- NG! 空白文字

特に日本語が利用できないことに留意して下さい。

```{tip}
このルールは実用上バランスがよく、かつ覚えやすいものです。
使える文字をこれより増やすことも可能ですが、覚える労のわりに出番が少なかったりするため割愛します。
```

例)
index.html
index.HTML
Index.html
INDEX.html
abc.html
123.html
0AbC.html
abc/.html
_abc_.html
-ab-c.html
2022/10/17.html
	記号は-_しか使えない
2022-10-17.html
2022 10 17.html
	スペース文字は使えない
2022　10　17.html
	(全角空白を使った)日本語は使えない
日本語.html
	日本語は使えない

index.html: このファイルには、一般的にあなたのホームページの内容、つまりあなたが最初にあなたのサイトに行ったときに見るテキストと画像が含まれています。テキストエディターを使用して、 index.html という名前の新しいファイルを作成し、 test-site フォルダー内に保存してください。
